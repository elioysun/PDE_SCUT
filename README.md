# PDE_SCUT

这是一个面向《数学物理方程 / PDE》期末复习的公开资料仓库。

仓库内容以复习可用为主，尽量保持轻量：主要放稳定笔记和作业整理，不追求收集所有资料。

## 仓库内容

| 路径 | 内容 |
| --- | --- |
| `notes/` | 章节笔记、考试要点、方法总结和思维导图。 |
| `raw/作业/` | 按章节整理的作业题和部分参考答案。 |

推荐阅读顺序：

1. 先看 `notes/数学物理方程考试要点.md`，快速了解复习重点。
2. 再看对应章节笔记，例如 `notes/第03章-分离变量法.md`。
3. 最后回到 `raw/作业/` 做题和对答案。

## 如何把仓库下载到本地

下面默认你已经安装了 Git。如果还没有安装，可以先搜索并安装 `Git`，安装后在终端输入：

```bash
git --version
```

如果能看到版本号，说明 Git 可以使用。

### 第一次 Clone

找一个你想存放资料的文件夹，在终端运行：

```bash
git clone https://github.com/elioysun/PDE_SCUT.git
cd PDE_SCUT
```

执行完后，本地会出现一个 `PDE_SCUT` 文件夹，里面就是仓库内容。

## 推荐用 Obsidian 打开

这个仓库已经按 Obsidian 笔记库的方式组织，推荐直接用 Obsidian 打开整个 `PDE_SCUT` 文件夹。

打开方式：

1. 启动 Obsidian。
2. 选择 `Open folder as vault` / `打开本地文件夹作为库`。
3. 选择刚刚下载的 `PDE_SCUT` 文件夹。

这样可以更方便地阅读 `notes/` 里的章节笔记、考试要点和思维导图，也能正常使用笔记之间的内部链接。

## 如何更新本地内容

如果你只是阅读资料，没有自己改文件，每次想同步最新内容时，在 `PDE_SCUT` 文件夹里运行：

```bash
git pull
```

常见流程是：

```bash
cd PDE_SCUT
git pull
```

这样就会把 GitHub 上的新内容更新到你的电脑。

## 如果你改过本地文件

如果你自己改过本地文件，再运行 `git pull` 时，可能会遇到冲突或提示你先处理本地修改。

最简单的做法是：如果只是做个人笔记，建议先把自己的修改复制到仓库外的私人笔记里，再执行：

```bash
git pull
```

如果你熟悉一点 Git，也可以先提交自己的修改：

```bash
git status
git add 文件名
git commit -m "Add my notes"
git pull
```

如果只是临时改动，想先收起来再更新：

```bash
git stash
git pull
git stash pop
```

`git stash pop` 之后如果出现冲突，需要手动选择保留哪一部分内容。

## 常用 Git 命令

| 命令 | 用途 |
| --- | --- |
| `git status` | 查看当前有哪些文件被修改。 |
| `git pull` | 从 GitHub 拉取最新内容。 |
| `git log --oneline -5` | 查看最近 5 次更新。 |
| `git restore 文件名` | 放弃某个文件的本地修改。 |

使用 `git restore 文件名` 会丢掉该文件的本地改动，执行前请确认不需要这些修改。

## 不会用 Git 怎么办

如果暂时不想安装或学习 Git，也可以在 GitHub 页面点击：

```text
Code -> Download ZIP
```

这样可以直接下载一份压缩包。

但这种方式不能自动更新。以后仓库内容更新时，需要重新下载 ZIP。

## 参与修改

如果你发现笔记里有错误，或者想补充更好的解法，可以通过 GitHub 提 issue 或 pull request。

提交修改时建议：

- 一次只改一个主题，不要把很多无关修改混在一起。
- 公式、答案或解法修改最好说明依据。
- 不要上传 `.DS_Store`、本地软件配置、私人笔记或无关文件。
